.או בסיום מיידי של התוכנית עם הודעה ברורה של סיבת השגיאה

	b. בדיקה אם יש הרשאת קריאה.
	c. בדיקה אם ה - Json תקין תחבירית
	d. בדיקה אם Decerialization לא מחזיר null.
	e. בדיקה אם הקובץ מחזיר מערך ריק.

אשר:

אחראי על model Drone
פלוס וולידיציות על כל שדה ושדה.

בניית interface של וולידציה.

ורשימת אובייקטים תקינים ולא תקינים.

* **מודל הנתונים (`Drone`):** מחלקה "טיפשה" שרק שומרת נתונים, כדי לשמור על עקרון האחריות היחידה (SRP).
* **מנוע הבדיקות (`DroneValidator`):** 
  * **מתודות קטנות:** חילקתי את הבדיקה לפונקציות עזר פרטיות שכל אחת בודקת חוק אחד.
  * **Guard Clauses:** ברגע שתנאי נכשל מחזירים מיד `false`. שומר על קוד קריא בלי `if` מקוננים.
  * **הכמסה (Encapsulation):** כל המספרים החשובים שמורים כקבועים פרטיים (`private const`) כדי למנוע תלויות.
* **טיפול בערכים מותרים:** שימוש ב-Enum למניעת שגיאות כתיב. עבור הדגמים (מכילים מקף שאינו חוקי ב-Enum), יצרתי מחלקה סטטית כחלופה מקצועית.
* **Regex (ביטוי רגולרי):** בדיקה קפדנית ויעילה שהמספר הסידורי עומד בדיוק בפורמט `DR-XXXX`.
